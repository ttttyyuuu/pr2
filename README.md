# 10-13 Практическое занятие 2 Разработка статического веб-интерфейса с применением принципов семантической разметки и адаптивного дизайна (Version 1.0)

## О демонтрационном проекте
Статический веб-интерфейс для магазина компьютерных комплектующих, разработанный с применением принципов **семантической разметки** и **адаптивного дизайна**.

### Описание структуры:
- **Папка `images`** содержит все изображения, используемые в ходе создания проекта;
- **Папка `pages`** представляет собой набор страниц приложения (авторизация и аутентификация);
- **Файл `index.js`** - главный файл и главная страница проекта;
- **Файл `.gitignore`** нужен для добавления файлов, которые не должны попадать в Git-репозиторий;
- **Файл `README.md`** - файл для документации проекта.

## Документация для работы с проектом
### Основные ресурсы:
- **[Tailwind CSS](https://tailwindcss.com/)** - утилитарный CSS-фреймворк, который позволяет стилизовать интерфейс с помощью готовых классов прямо в HTML или JSX/TSX. В отличие от традиционных CSS-фреймворков, Tailwind не предоставляет готовых компонентов, а даёт набор низкоуровневых утилитарных классов;
- **[Google Fonts](https://fonts.google.com)** - получение шрифтов;
- **[Font Awesome v4](https://fontawesome.com/v4/icons/)** - иконки для приложения;
- **[FontShare](https://www.fontshare.com/)** - облачный сервис по доставке шрифтов;
- **[MDN HTML](https://developer.mozilla.org/en-US/docs/Web/HTML)** - HTML документация от MDN;
- **[MDN CSS](https://developer.mozilla.org/en-US/docs/Web/CSS)** - CSS документация от MDN;
- **[git](https://git-scm.com/)** - скачивание Git к себе на устройство, чтобы пользоваться консолью и интерфейсом;
- **[node.js](https://nodejs.org/en/download)** - установка Node.js на устройство (+ npm, python и другие важныеи инструменты);
- **[Dribble](https://dribbble.com/)** - для идей также используем Dribble;
- **[Tailwind CSS Showcase](https://tailwindcss.com/showcase)** - примеры проектов с использованием Tailwind CSS.

## Запуск проекта
Для запуска проекта необходимо в среде разработки VS Code установить расширение **Live Server**.

### Шаги для запуска:
1. **Установите расширение Live Server** в VS Code
2. **Запустите проект** через Live Server (не из .html-файла напрямую, а на порте)

<img width="756" height="579" alt="Инструкция по запуску Live Server" src="https://github.com/user-attachments/assets/4a32d46b-bdea-499d-8099-85c898d2edb0" />

### Преимущества использования Live Server:

- локальный сервер для тестирования;
